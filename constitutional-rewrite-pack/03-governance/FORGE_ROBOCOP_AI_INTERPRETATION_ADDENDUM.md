# ROBOCOP ADDENDUM 001 - Forge Thinks, AI Interprets

Status: LOCKED GOVERNANCE ADDENDUM

Scope: AI interpretation boundary

Implementation: Documentation only

Code authority: None

## Prime Directive

Forge thinks. AI interprets.

AI SHALL NOT OVERRIDE FORGE.

No AI system, LLM, assistant, prompt, model, completion, embedding, summary, classification, or generated recommendation may override Forge's Constitution, ADRs, source truth, rule engines, authority gates, Board boundaries, Miranda approval, or ROBOCOP LOCK.

AI is an interpretation layer.

Forge is the decision system.

## Authority Order

All AI-assisted behavior must obey this authority order:

1. Forge Constitution
2. Locked / Final / Operational ADRs
3. ROBOCOP LOCK 001
4. Build Tree status
5. Discovery status and implementation readiness
6. Source truth and evidence packets
7. Rule engines and domain engines
8. Forge Orchestrator
9. Authority gates and permission boundaries
10. AI interpretation layer
11. Presentation / explanation / drafting layer

If AI output conflicts with any higher authority, the AI output is invalid.

## AI Output Classification

AI output is not source truth.

AI output must never be treated as:

- FACT
- SOURCE_OF_TRUTH
- RULE
- POLICY
- DECISION
- OFFICIAL_METRIC
- PRODUCT_TRUTH
- FORECAST_TRUTH
- COMPENSATION_TRUTH
- MANAGER_JUDGMENT
- CLIENT_INTENT
- HUMAN_VALUE_JUDGMENT

AI output may only be classified as:

- AI_INTERPRETATION
- AI_SUMMARY
- AI_DRAFT
- AI_EXPLANATION
- AI_EXTRACTION_CANDIDATE
- AI_CLASSIFICATION_CANDIDATE
- AI_LANGUAGE_HELPER
- AI_PRESENTATION_HELPER

Any AI-generated interpretation must be validated by Forge before it becomes a recommendation, action, insight, status, score, or user-facing decision support.

## Required Forge Validation

Forge validates before action.

Before any AI output can influence a recommendation or action, Forge must validate:

- Evidence source
- Evidence ownership
- Provenance
- Unknowns
- Applicable ADRs
- Product truth boundaries
- Forecast truth boundaries
- Recommendation vs decision authority
- Advisor OS / Manager OS boundary
- User permissions
- Miranda approval
- Board boundary, when applicable

If validation fails, Forge must block the output or downgrade it to a clearly labeled interpretation.

## Alfred Boundary

Alfred is the Universal Search, Index and Command Interface of Forge OS.

Alfred may use AI to interpret natural language, summarize results, draft text, or explain context.

Alfred must not bypass Forge.

For search/index behavior, Alfred must follow:

```text
User input
-> Forge Index
-> Permission Filter
-> Context Builder
-> Authority Gate
-> Forge Orchestrator
-> Engine Registry
-> AI interpretation, only if needed
-> Forge validation
-> Result / explanation / action
```

For command behavior, Alfred must follow:

```text
User intent
-> Intent Detection
-> Scope Detection
-> Permission Check
-> Command Authority Gate
-> Forge Orchestrator
-> Required confirmation, if mutating data
-> AI drafting/explanation, only if needed
-> Forge validation
-> Action or blocked state
```

Alfred cannot execute governed actions directly from AI output.

## Examples

Allowed:

User types:

```text
/Carlos
```

Forge searches the user's indexed database, applies permissions, ranks real entities and contexts, then Alfred displays:

- Carlos Sanchez - Prospecto - Cita de cierre
- Carlos Martinez - Cliente - ORVI - Poliza xxxxx
- Carlos Rodriguez - Seguimiento - Llamar miercoles

AI may help summarize why one result appears first, but only after Forge ranking and context validation.

Blocked:

User types:

```text
Who should I ignore this week?
```

AI may not decide this directly.

Forge must evaluate action ownership, evidence, relationship status, Advisor OS boundaries, Manager OS boundaries, and non-manipulation rules before any recommendation.

Blocked:

AI says:

```text
Carlos is low value.
```

Invalid.

This is a human value judgment and may not be treated as Forge truth.

Allowed alternative:

```text
Carlos has no open opportunity, no recent interaction, and no advisor-owned next action based on current evidence.
```

## Manager OS Boundary

In Manager OS, AI may summarize or explain signals.

AI may not autonomously judge human worth, potential, loyalty, discipline, coachability, or future success.

Andrey / Human Capital Allocation signals must be governed by evidence, RODI boundaries, Miranda approval, and Manager OS authority.

AI cannot create or expose Manager OS judgments inside Advisor OS.

The Miranda Wall remains active.

## Final Rule

Forge thinks. AI interprets.

When in doubt:

- Forge must ask for evidence.
- Forge must preserve boundaries.
- Forge must classify uncertainty.
- Forge must block unauthorized action.
- Forge must never allow AI to become source truth.

If AI tries to override Forge, the task is:

```text
BLOCKED BY ROBOCOP LOCK 001.
```
