# Technical Specification Guide

Status: Ratified

## Purpose

Technical Specifications define HOW an approved architecture must be implemented.

## Specifications may define

- APIs
- Interfaces
- Contracts
- Data models
- Protocols
- Message formats
- Validation rules
- Runtime behavior
- Error handling
- Security requirements
- Performance expectations

## Specifications SHALL NOT

- redefine governance
- replace ADRs
- introduce architecture
- contradict the Constitution

## Relationship

Constitution
↓
Axioms
↓
Governance
↓
