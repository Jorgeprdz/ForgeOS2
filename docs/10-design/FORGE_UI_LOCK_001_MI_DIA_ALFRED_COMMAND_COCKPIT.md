# FORGE UI LOCK 001 — Mi Día / Alfred Command Cockpit

Status: LOCKED  
Date: 2026-06-19  
Scope: Advisor mobile opening experience  
Surface: Forge OS mobile app  
Screen: Mi Día  
Primary intelligence layer: Alfred  

---

## 1. Lock Statement

This document locks the approved visual and experiential direction for the Forge OS advisor home screen.

The locked screen is:

FORGE UI LOCK 001 — MI DÍA / ALFRED COMMAND COCKPIT

This screen defines how Forge OS should feel when an advisor opens the app.

Forge must not feel like:

- a generic CRM
- a corporate dashboard
- a database UI
- a report viewer
- a spreadsheet wrapper

Forge must feel like:

A premium commercial operating system that tells the advisor what to do next.

---

## 2. Product North Star

Forge OS exists to answer one question:

¿Qué debo hacer hoy para cerrar más negocios, ayudar mejor a mis clientes y construir una carrera más rentable?

The advisor should not open Forge to interpret dashboards.

The advisor should open Forge and immediately know:

- where they stand
- what matters
- what is at risk
- what opportunity to move
- what action to take now

The emotional outcome is:

Ya sé exactamente qué hacer.

---

## 3. Strategic Positioning

Forge OS is the Salesforce Killer for financial advisors and insurance sales teams.

This does not mean Forge copies Salesforce.

It means Forge competes by rejecting generic CRM complexity.

Salesforce tracks work.  
Forge tells advisors what to do next.

Salesforce shows dashboards.  
Forge produces next best action.

Salesforce stores records.  
Forge turns data, conversations, relationships, policies, behavior and metrics into commercial action.

---

## 4. Locked Visual DNA

The approved visual direction is:

- premium mobile-first interface
- iOS / One UI inspired polish
- dark elegant glassmorphism
- soft blur
- rounded layered cards
- subtle glowing accents
- calm but powerful hierarchy
- high contrast, readable typography
- executive, modern, emotional
- action-oriented, not analytics-heavy

The UI must feel premium, not playful.
It must feel intelligent, not decorative.
It must feel decisive, not informational.

---

## 5. Contextual Header Lock

The screen must include a contextual visual header.

The header may adapt to:

- time of day
- weather
- local context
- mood of the day
- advisor state

Examples:

- morning sunrise
- cloudy morning
- rainy day
- late afternoon focus mode
- evening wrap-up

The contextual image must feel integrated into the UI, not like a wallpaper pasted behind cards.

Required header elements:

- greeting: Buenos días, Jorge
- screen title: Mi día
- contextual weather/time capsule when available
- elegant ambient image or scene

The contextual header is important because Forge should feel alive and aware.

---

## 6. Alfred Lock

Alfred is not an optional assistant.

Alfred is the operational voice of Forge.

Alfred must be visible on the opening screen.

Alfred represents:

- interpretation
- prioritization
- commercial judgment
- next best action
- daily briefing
- command execution

Alfred should feel like:

- a refined commercial butler
- a strategic copilot
- a calm operating partner
- not a toy chatbot
- not a generic AI bubble

The Alfred briefing card is a primary screen element.

Required Alfred card content pattern:

Alfred

Hoy tu cuello de botella es seguimiento.  
Empieza por Lariza antes de las 11:00.

Button: Iniciar plan de hoy

Alfred should not merely answer questions.

Alfred should guide the advisor into action.

---

## 7. Command Bar Lock

The Alfred command bar is required.

It must be visible and persistent on the opening experience.

The command bar communicates:

You can ask Alfred what to do now.

Approved command bar behavior:

- sits near the lower part of the screen, above bottom navigation
- pill-shaped glass input
- subtle glow
- Alfred icon on the left
- microphone and send/action icon on the right
- feels like a premium command interface, not a chat box

Suggested placeholder:

Pídele a Alfred qué hacer ahora

Alternative for top command usage:

Pregúntale a Alfred o escribe un comando...

The command bar is part of the product identity.

Forge is not just navigated.  
Forge is commanded.

---

## 8. Required Screen Sections

The locked Mi Día screen includes these sections:

### 8.1 Alfred Briefing

Purpose:
Tell the advisor the most important interpretation of the day.

Example:

Hoy tu cuello de botella es seguimiento.  
Empieza por Lariza antes de las 11:00.

### 8.2 Forecast / Goal Card

Purpose:
Show commercial state in one glance.

Required elements:

- Probabilidad de alcanzar meta mensual: 78%
- Producción esperada: $142,000
- Gap: $58,000
- Riesgo principal: 3 clientes sin seguimiento > 14 días

This card should not become a dashboard dump.
It must summarize state and risk.

### 8.3 Oportunidades

Purpose:
Show priority opportunities, not a raw prospect list.

Example:

- Lariza 72%
- Octavio 65%
- María 40%
- José 15%

Each opportunity should imply urgency and likelihood.

### 8.4 Motor de Seguimiento

Purpose:
Expose neglected opportunities and rescue actions.

Example:

- Cliente olvidado
- Último contacto: 18 días
- Probabilidad de rescate: 64%
- Sugerencia: Enviar mensaje de reconexión

### 8.5 Alfred Recomienda

Purpose:
Translate analysis into a short action list.

Example:

1. Llamar a Lariza
2. Rescatar a Marlene
3. Agendar 2 citas

This is the advisor’s operating list for the day.

## 9. Bottom Navigation Lock

Required bottom navigation:

- Inicio
- Pipeline
- Clientes
- Alfred
- Más

Alfred must be visually special.

Approved treatment:

- highlighted Alfred tab
- subtle glow
- refined icon
- not cartoonish
- associated with intelligence / command

The nav should support the product hierarchy:

- Inicio = Mi Día / next action
- Pipeline = opportunities
- Clientes = relationship and policy context
- Alfred = command / briefing / assistant
- Más = secondary tools

---

## 10. Forbidden Drift

The Mi Día screen must not become:

- a wall of metrics
- a generic CRM homepage
- a Salesforce-style dashboard
- a list of modules
- a BI report
- a table-first interface
- a page that requires the advisor to interpret too much

If a new widget does not help answer “what should I do next?”, it does not belong on Mi Día.

---

## 11. Data-to-Action Principle

The locked flow is:

Data -> context -> risk -> priority -> next best action

Not:

Data -> charts -> dashboard -> user must interpret

Forge must reduce cognitive load.

---

## 12. Advisor Experience Standard

After using Forge, the advisor should say:

Ya no tengo que pensar en qué hacer después.

This is the primary UX test.

---

## 13. Manager Experience Alignment

Although this lock is for the advisor opening screen, the same principle applies to Manager OS.

The manager should not say:

Tengo muchos reportes.

The manager should say:

Sé exactamente a quién ayudar, con qué ayudarlo y cuándo intervenir.

Manager OS must also convert data into intervention.

---

## 14. Implementation Guidance

Future UI implementation must preserve:

- Alfred-first hierarchy
- Alfred as visible operational layer
- command bar as a first-class control
- contextual header based on time, weather or advisor state
- forecast and gap as concise decision signal
- opportunities ranked by priority and probability
- follow-up engine as rescue layer
- recommendations as action list
- premium dark glass visual language

Any UI implementation that removes Alfred, hides the command bar, buries the daily briefing, or turns Mi Día into a generic dashboard violates this lock.

FORGE UI LOCK 001 — MI DÍA / ALFRED COMMAND COCKPIT
