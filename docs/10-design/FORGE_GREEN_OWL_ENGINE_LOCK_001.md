# FORGE GREEN OWL ENGINE LOCK 001

Status: LOCK CANDIDATE
Scope: Advisor activity, streaks, challenges, nudges, manager diagnosis and behavioral momentum
Primary owner: Mick
Supporting systems: Alfred, Candy Crush Experience, Rocky, Nash, Manager OS

## 1. Lock Statement

The Green Owl Engine is Forge OS behavior loop for daily commercial activity.

It converts advisor activity into daily points, streaks, adaptive challenges, motivational nudges, Chrome/PWA notifications, daily confirmation, manager diagnosis and conscious next actions.

Forge must not turn activity into a static dashboard.
Forge must turn activity into visible commercial progress.

## 2. Core Product Principle

Forge must not make the advisor work more in order to use Forge.
Forge must reduce friction.

Canonical rule:
Forge debe sentirse como atajo, no como tramite.

## 3. Minimum Clicks Principle

Every important action should have the shortest possible path.

Required paths:
1. Command bar path
2. Shortcut path
3. Traditional navigation fallback

Examples:
/Mensaje a Juan
/Agregar poliza
/Registrar pago
/Agendar con Luis miercoles 5 pm
/Bitacora Luis
/Registrar 3 llamadas

Canonical rule:
Command-first, navigation-second.

## 4. Daily Activity Points

The Green Owl Engine operates on top of Forge activity points.

Current activity categories include referidos, llamadas, citas agendadas, citas iniciales, citas de cierre, solicitudes and polizas pagadas.

The scoring system must have one source of truth.
Mick owns behavioral interpretation.
No module should calculate official activity points with conflicting scoring rules.

## 5. Explain-the-Why Nudge Rule

Green Owl must never issue blind commands.
Every nudge must explain the reason and expected impact.

Bad:
Haz 3 llamadas mas.

Good:
Haz 3 llamadas mas y puedes generar hasta 12 puntos adicionales: 3 por llamadas y 9 si se convierten en citas agendadas.
Con eso cierras el dia arriba de 25 y mantienes tu racha.

Canonical rule:
Forge no empuja obediencia. Forge genera conciencia operativa.

## 6. Daily Activity Confirmation Loop

At 8:00 pm, Forge should ask the advisor to confirm the day.

Example:
Hoy registre: 3 referidos, 2 citas iniciales, 1 cita de cierre. Tuviste o tienes mas actividad?

Actions:
Confirmar
Editar actividad
Agregar mas

After confirmation:
- close daily points
- update streak
- update activity history
- trigger manager summary when applicable

Canonical rule:
La actividad diaria no queda cerrada hasta que el asesor la confirma.

## 7. Manager Activity Diagnosis

The 25-point system must also help the manager.
Manager OS must not only show totals.
It must identify where the advisor is failing in the funnel.

Examples:
Juan tiene muchas citas iniciales, pero pocas de cierre. Diagnostico: falla en avanzar discovery hacia propuesta/cierre.
Pamela agenda muchas citas, pero no tiene citas iniciales reales. Diagnostico: falla entre agendamiento y asistencia.
Rodrigo no obtiene referidos. Diagnostico: falla en activar red / pedir referidos.

Canonical rule:
No me digas quien va abajo. Dime por que va abajo y que conversacion debo tener.

## 8. Calendar-to-Activity Confirmation Loop

Forge should reduce manual capture by connecting with Google Calendar.

Example:
Agende con Luis el miercoles a las 5.

Forge should create the Google Calendar event, link it to Luis, classify it as commercial activity and schedule post-meeting confirmation.

After the meeting Forge asks:
Si tuviste la cita con Luis?

Options:
Si, se realizo
No se presento
Reagendar
Agregar nota

Calendar is evidence of scheduled activity.
Post-meeting confirmation is evidence of completed activity.

## 9. Meeting Bitacora / Meeting Memory

After a confirmed meeting, Forge should ask: Cuentame que paso.

Advisor input may be text or voice.

Example:
Luis se vio interesado, es soltero, le interesa ahorro, tiene 24000 de presupuesto, lo vere el viernes.

Forge should structure prospect, interest level, need, budget, objections, next meeting, next action, pipeline stage and follow-up reminder.

Canonical rule:
La bitacora no se captura. La bitacora se cuenta.

## 10. Ephemeral Voice Cache Rule

Voice notes may be used to reduce friction.
Audio must be temporary.

Forge may cache voice only long enough to transcribe, extract structured context, let the advisor confirm or edit, and save structured meeting memory.

Then the audio must be deleted.

Durable records may include approved transcript if needed, structured bitacora, confirmed activity, next action and pipeline update.
Durable records must not include raw audio by default.

## 11. Advisor Pattern Learning

Confirmed bitacoras should help Forge understand advisor patterns.

Forge may learn common client segments, frequent needs, common budgets, repeated objections, conversion bottlenecks, product fit, message effectiveness and advisor behavior patterns.

Each bitacora teaches two things: what the prospect needs and how the advisor sells.

## 12. Forge Thinks, AI Transforms

Forge owns reasoning.
AI only transforms.

Forge owns points, streaks, scoring, recommendations, pipeline state, forecast, activity diagnosis, rule packs, source truth and commercial decisions.

AI may support transcription, extraction, summarization, message drafting, explanation, ambiguity handling and natural language to structure.

Canonical rule:
Forge thinks. AI transforms.

## 13. Minimal AI Consumption

AI usage must be minimal, selective and cost-aware.

Do not call AI for point calculation, streak calculation, basic gap detection, rule pack logic, commission calculations, deterministic scoring, simple notifications or source truth.

Use AI only where language or ambiguity makes it valuable.

Canonical rule:
Every AI call must justify its cost.

## 14. WALL-E Compliance

Green Owl must strengthen human judgment, not replace it.
Nudges should help the advisor understand why an action matters.

Forge must not create blind obedience, dependency loops, shame loops, surveillance loops, black-box behavior or manager coercion.

Human agency stays above automation.

## 15. Evidence First Compliance

Green Owl must not invent.

Evidence rules:
- Calendar event = scheduled activity evidence
- Post-meeting confirmation = completed activity evidence
- Confirmed bitacora = commercial memory evidence
- Temporary audio = input, not source truth
- AI extraction = interpretation, not source truth
- Manager summary = based on confirmed activity, not guesses

Canonical rule:
Evidence First. Unknown remains unknown. Forge never invents.

## 16. Forbidden Drift

Green Owl must not become vanity gamification, childish badges without business value, pressure without context, shame-based reminders, generic productivity nagging, manager surveillance, blind activity obsession or AI-driven invented motivation.

If a challenge does not help the advisor move toward better commercial behavior, it does not belong.

Forge no castiga al asesor por ir abajo.
Forge le da el siguiente reto que si puede ganar.

Alfred decide el siguiente movimiento.
Mick entrena la conducta.
Green Owl protege la racha.
