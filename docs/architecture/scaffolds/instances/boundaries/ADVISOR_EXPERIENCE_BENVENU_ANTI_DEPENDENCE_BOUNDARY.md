# Advisor Experience and Benvenù Anti-Dependence Boundary
## Purpose
Ensure explanation, help and first value strengthen rather than replace advisor judgment.
## Scope
Contextual help, learning states, progressive discovery, first value, capture and interruption limits.
## Responsibilities
Require capability growth; preserve upstream truth; stop repetitive/irrelevant help; create value before manual work.
## Authority
Article 0 / Ley Zero; ADR-016; ADR-020; SG-002 Gate. Owner evidence: ADR-016.
## Boundaries
Permitted: explanation, learning support and first value. Prohibited: dependency, blocking work, duplicate capture, upstream truth/recommendation ownership. Advisor retains judgment; failure is `UNKNOWN / BLOCKED`.
## Dependencies
Official outputs, evidence/learning state and workflow context; no reverse authority into source domains.
