# Advisor Experience and Benvenù Domain Responsibility

## Purpose
Document Advisor Experience with nested Benvenù responsibilities that strengthen capability and create first value without dependency.
## Scope
Advisor-facing explanation, contextual help, feature-learning state and progressive discovery; Benvenù owns the first-value experience within this domain.
## Responsibilities
Build comprehension and autonomy; adapt help to evidence and learning state; deliver first value before heavy capture; suppress irrelevant help.
## Authority
Article 0 / Ley Zero; Unified Constitution Articles IV–VII; ADR-016; ADR-020; SG-002 Gate. Owner evidence: ADR-016 establishes Advisor Experience and nested Benvenù owners.
## Boundaries
Does not own upstream truth, recommendations, UI design, implementation logic or human decisions. Help may not block work, cultivate dependency or require duplicate capture.
## Dependencies
Official domain outputs, evidence/unknown states, user role, workflow context and feature-learning evidence.

Owners: Advisor Experience for advisor-facing explanation/learning state; Benvenù for first-value experience. Registrations: [Advisor Experience](../source-truth/ADVISOR_EXPERIENCE_OWNERSHIP.md) and [Benvenù](../source-truth/BENVENU_FIRST_VALUE_OWNERSHIP.md).
