# ADR-0025: Habit Intelligence Boundary

## Status
CANDIDATE

## Context
The EFC manual states that prospecting is a habit.

This reframes a major advisor failure mode.

Commercial failure is often not lack of knowledge.

It is lack of consistency.

Forge already recognizes Mick / Behavior Intelligence as the boundary for habits, discipline, consistency, coachability, activity generation and execution patterns.

## Decision
Habit Intelligence is the Advisor OS and Manager OS bridge for behavior consistency.

Habit Intelligence should eventually track:

- prospecting habits
- calling habits
- follow-up habits
- activity cadence
- avoidance patterns
- consistency breaks
- behavior recovery signals

Mick / Behavior Intelligence should eventually govern this boundary.

## Boundaries
Habit Intelligence measures behavior that creates future outcomes.

It does not replace performance metrics.

It does not own compensation.

It does not shame the advisor.

It should produce actionable coaching or execution signals.

Advisor OS consumes habit signals to help the advisor act.

Manager OS consumes habit signals to help the manager coach.

## Consequences
Positive:

- Forge can detect broken commercial behavior before production collapses.
- Advisor execution and manager coaching can share evidence without sharing ownership.
- Activity can be interpreted as habit health, not merely volume.

Negative:

- Habit signals can become harmful if presented without context, confidence or actionability.
- Manager-facing interpretations must avoid replacing human judgment with surveillance.

Neutral:

- This ADR candidate does not implement Mick.
- This ADR candidate does not change activity route behavior.
