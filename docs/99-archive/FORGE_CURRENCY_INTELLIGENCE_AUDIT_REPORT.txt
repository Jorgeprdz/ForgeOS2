6. IMAGINA SER IMPACT

What currency-related work is already done?

- UDI is recognized as the core Imagina Ser currency in tests and presentation flow.
- Imagina Ser master test validates UDI currency and UDI scenario values.
- Imagina Ser retirement fund analysis calculates UDI contribution, final reserve, growth and growth percent.
- Retirement presentation scenario converts UDI summary values to MXN when a currentUdiValue is supplied.
- General Forge currency infrastructure exists:
  - Live Banxico UDI retrieval.
  - Live Banxico USD/MXN FIX retrieval.
  - Local 12-hour exchange rate cache.
  - Explicit currency normalization for MXN/USD/UDI.
  - UDI and USD scenario projection.
  - Dynamic cash value projection using OCR rows.

What is still missing before Imagina Ser can be considered closed?

1. Wire Imagina Ser to the market data layer.
- Current Imagina Ser / retirement flows use explicit or default currentUdiValue values such as 8.7.
- No evidence found that imagina-ser-master-test.js, imagina-ser-client-presentation-engine.js or retirement-presentation-scenario-engine.js obtain current UDI through getCachedRates.

2. Remove or isolate hardcoded retirement quote values.
- solucionline-retirement-parser.js hardcodes basicAnnualPremium, plannedAnnualContribution, totalAnnualPremium, sumAssured and scenario values.
- For closure, those values should come from source document extraction or explicit evidence, not embedded constants.

3. Add source/evidence metadata to Imagina Ser MXN conversions.
- Currency outputs should identify rate source, rate date, cache/live status and confidence.
- Existing Banxico/cache payload supports this, but Imagina Ser presentation outputs do not appear to carry it through.

4. Clarify current-rate vs future-scenario language in Imagina Ser outputs.
- Forge has calculationMode labels such as SCENARIO_BASED_ESTIMATE_NOT_GUARANTEED and MXN_TIMELINE_SCENARIO_NOT_GUARANTEED in shared modules.
- Imagina Ser should consistently expose similar labels when showing MXN equivalents or future values.

5. Add Imagina Ser-specific tests for Banxico-backed UDI.
- Existing tests cover Banxico/cache separately and Imagina Ser separately.
- Missing: an Imagina Ser acceptance test proving that current UDI can be sourced from cache/Banxico and used in presentation/retirement outputs without hardcoded 8.7.

6. Decide whether inflation is UDI-only or a separate domain.
- UDI projection exists.
- A distinct Inflation Projection domain was not found.
- If Imagina Ser requires explicit inflation intelligence, it is not closed yet.
