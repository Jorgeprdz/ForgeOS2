# Codex Boundary

Codex defines and prepares the product system, contracts, scaffolds and execution machinery.

Codex does not implement Forge OS 2.

The functional rewrite, operational validation, commits, pushes and promotion are performed later from Termux under owner control.

Codex-created artifacts:

- `docs/product/FORGE_PRODUCT_SPEC.md`
- `docs/product/FORGE_REQUIREMENTS_TRACEABILITY_MATRIX.md`
- `scaffolds/manifest/`
- `scaffolds/contracts/`
- `scaffolds/templates/`
- `scaffolds/validation/`
- `tools/termux/rewrite/`

Codex-prohibited actions for this scaffold task:

- final screens;
- production business logic;
- engines;
- service integrations;
- Supabase production migrations;
- secrets;
- legacy code copy;
- main promotion.
