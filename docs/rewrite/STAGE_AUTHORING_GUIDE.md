# Stage Authoring Guide

Each stage must declare:

- stable `SG-###` identifier;
- product capabilities;
- functional requirements;
- constitutional authority;
- applicable boundaries;
- ADR or architectural decision;
- contracts;
- prerequisites;
- allowed and prohibited paths;
- templates;
- files to generate;
- validations;
- evidence;
- acceptance criteria;
- rollback rule;
- status.

Do not mark a stage `READY` unless every capability has traceability and the stage generates only authorized files.
