# Forge OS Constitutional Rewrite Pack

Status: UNIFIED CONSTITUTIONAL BASELINE / NO IMPLEMENTATION AUTHORITY

## Purpose

This pack is the minimum self-contained documentary baseline for a constitutional rewrite of Forge OS. It does not authorize implementation. It preserves the authority chain, canonical ADRs, governance gates, master Build Tree and the only current conceptual truth-validation contract retained for rewrite analysis.

## Authority order

The active Source of Truth is root `CONSTITUTION_UNIFIED.md`. Article 0/Ley Zero is its first article. Active axioms are indexed by root `governance/axioms/INDEX_V2.md`. SKYNET remains unratified. Human judgment, evidence, unknown-state, ROBOCOP and Miranda boundaries remain mandatory. The Constitutional Freeze remains active for software implementation until the remaining contracts in `REWRITE_PREREQUISITES.md` are ratified.

- `10-unification-execution/`: authorization failure history, successful execution evidence, changelog, traceability, diff and final validation.
- `INVENTORY.md`: complete provenance and dependency index.
- `VALIDATION_REPORT.md`: redundancy, contradiction, exclusion and gap audit.
- `REWRITE_PREREQUISITES.md`: indispensable documents that must be ratified before implementation begins.

## Use restriction

This pack may be used for governance reconciliation, architecture definition, contract definition and rewrite readiness only. It must not be treated as permission to recover legacy implementations, archived domain models, draft specifications, scripts, tests, snapshots, backups or cleanup artifacts.

## Provenance rule

Files marked `canonical` come from the current repository unless the inventory explicitly says `historical constitutional recovery`. Historical recovery is limited to indispensable constitutional or governance authority missing from the cleaned repository. In conflicts, the current repository and its ratified Constitution/ADRs prevail.
