# Forge GMM Shared Source Review

Status: DISCOVERY / KNOWLEDGE GOVERNANCE

## Purpose

Determine which sources belong to the Shared GMM Layer versus product-specific
layers.

## Shared GMM Layer Sources

Candidate shared sources:

- Source governance standards.
- Evidence sufficiency standards.
- Human review standards.
- Assessment language standards.
- Generic source registry rules.
- Product routing governance.
- Source quality classification.
- Source conflict workflow.

These sources define how Forge thinks, not product truth.

## Product Layer Sources

Product-specific sources:

- Alfa Medical Condiciones Generales.
- Alfa Medical Flex Condiciones Generales.
- Product manuals.
- Product-specific optional coverage documents.
- Product-specific endorsements.
- Product-specific caratulas.
- Product-specific waiting-period lists.
- Product-specific benefit maps.
- Product-specific exclusions.

These sources define product truth.

## Shared Operational Sources

Potentially shared but date/product-sensitive:

- Hospital directories.
- Provider directories.
- Pharmacy lists.
- Honorarium catalogs.
- Usual/reasonable catalogs.

These may serve multiple products but must still be tied to:

- Product applicability.
- Plan.
- Zone.
- Date.
- Provider classification.

## Evidence Sources

Case-specific evidence sources are shared in concept but case-specific in use:

- Clinical file.
- Diagnosis.
- Prescription.
- Invoice.
- Authorization.
- Prior policy evidence.

## Shared vs Product Boundary

Shared GMM Layer may say:

"Coverage evaluation requires product identity, policy context, event family,
evidence sufficiency, human review gates and source provenance."

Shared GMM Layer must not say:

"Maternity is covered after X months" unless the product-specific source
supports that for the product in question.

## Verdict

Shared sources govern evaluation discipline.

Product sources govern coverage truth.
